Инструкция по работе с git


Инициализация

git init - инициализация гита в репозитории

Статус

git status - проверить состояние гита

Отслеживание файлов гитом

git add полноеНазваниеФайла - добавить файл в отслеживаемые (необходимо вводить перед каждым коммитом)
git add . - добавить в отслеживаемые все файлы в каталоге

Коммиты

git commit -m "комментарий" - произвести коммит. Позволяет сохранить изменения в новой версии

Работа с прошлыми версиями проекта

git log - просмотреть список всех коммитов.
Флаг --oneline - один коммит выводится в одну строку.
Флаг --graph - коммиты выводятся в виде дерева.
Эти флаги можно комбинировать.
git checkout номерВерсии - вернуться к версии
git diff - просмотреть разницу между текщим и последней версиями

Ветки

Ветвление стало неотъемлемой частью командной разработки, потому что оно дает возможность работать над разными версиями исходного кода. Основной идеей ветвления является отклонение от основного кода и продолжение работы независимо от него. Также это удобно в тестировании отдельного функционала, потому что позволяет работать над новой частью кода, не беспокоясь о поломке чего-то в рабочей версии.
git branch - отобразить список веток (показывает также текущуюю ветку)
git branch название ветки - создать новую ветку
git branch -M название ветки - переименовать
git branch -d название ветки - удалить ветку
git branch -D название ветки - принудительно удалить ветку

Слияние веток

git merge название ветки - слить ветку с текущей
